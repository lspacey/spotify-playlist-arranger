## STEP 2 — Module responsibilities and source mapping

### `config.py`
- Load `.env` via `python-dotenv`
- All constants from `playlist_analyzer.py`: `SAMPLE_RATE`, `MERT_SR`, `BUFFER_SECONDS`,
  `SPOTIFY_SCOPE`, `REDIRECT_URI`, `KEY_NAMES`, `CAMELOT`, `CAMELOT_TO_IDX`, `WEIGHTS`,
  `ARTIST_PENALTY`, `ALBUM_PENALTY`, `POLL_*`, `CHUNK`, `SEG_SECONDS`, etc.
- All LLM env vars: `LLM_BACKEND`, `OLLAMA_MODEL`, `DEEPSEEK_MODEL`, `MISTRAL_MODEL` and
  their API keys
- Configurable paths (with defaults relative to the script directory):
  - `DB_PATH`: `<root>/database/tracks_db.sqlite`
  - `EMBEDS_DIR`: `<root>/embeddings/`
  - `CACHE_DIR`: `<root>/cache/`
- A `Settings` dataclass with all the above paths + algo params (SA iterations, weights,
  duration tolerance), persisted to `cache/settings.json` via `atomic_write_json`
- Audio device selection: `SELECTED_AUDIO_DEVICE_INDEX` (default: first WASAPI loopback)

### `database/db.py`
- SQLite backend replacing `tracks_db.json`
- Schema: single table `tracks (spotify_id TEXT PRIMARY KEY, data TEXT NOT NULL)`
- `PRAGMA journal_mode=WAL` and `PRAGMA synchronous=NORMAL` on every connection
- Functions:
  - `get_track(track_id: str) -> dict | None` — single lookup by primary key
  - `save_track(track_id: str, entry: dict)` — atomic upsert
  - `load_all() -> dict` — returns `{id: dict}` for full DB scan (used by sorting)
  - `save_many(records: dict)` — bulk upsert via `executemany`
  - `delete_track(track_id: str)`
  - `vacuum()` — `EXECUTE VACUUM`
  - `analyze()` — `EXECUTE ANALYZE`
  - `track_count() -> int`
- Use a `threading.Lock` (same pattern as `_db_lock` in source) for write operations

### `database/migrate.py`
- `migrate_from_json(json_path: pathlib.Path)` — reads `tracks_db.json`, calls
  `db.save_many()`, prints migration stats

### `audio/capture.py`
- Port `list_loopback_devices()`, `start_audio_capture()`, `_make_callback()` verbatim
- Expose `audio_deque`, `full_buf`, `start_buf`, `start_buf_done` as module-level state
- Add `stop_capture(pa, stream)` helper

### `audio/features.py`
- Port `_extract_features()`, `_extract_features_full()`, `_trim_silence()`, `_to_mono()`
  verbatim from source
- No changes to logic

### `audio/mert.py`
- Port `load_mert()`, `_mert_embedding()` verbatim
- Model/extractor as module-level singletons with a `threading.Lock`

### `sources/spotify_source.py`
- Port `init_spotify()`, `get_own_playlists()`, `get_playlist_tracks()`,
  `play_track_on_device()`, `_spotify_request_with_retries()` verbatim
- Add `reorder_playlist(sp, playlist_id, ordered_uris)` — correct implementation:
  PUT first 100 URIs (full replace), then POST remaining chunks of 100 with
  `time.sleep(0.3)` between each, all via `_spotify_request_with_retries`
- Add `create_playlist(sp, name, uris)` — creates new playlist and adds tracks,
  also uses `_spotify_request_with_retries` and `time.sleep(0.3)` between chunk POSTs
- `is_configured() -> bool` — returns True if `SPOTIPY_CLIENT_ID` and
  `SPOTIPY_CLIENT_SECRET` are set in env

### `sources/local_source.py`
- `scan_folder(folder: pathlib.Path) -> list[dict]` — scans for `.mp3` and `.flac` files
- For each file, read ID3/FLAC tags using `mutagen` (add to requirements):
  `name` (TIT2 or filename stem), `artist` (TPE1), `album` (TALB), `duration_ms`
- `make_track_id(file_path: pathlib.Path) -> str` — compute `SHA1` of the first 512 KB
  of file bytes; this is the stable unique track ID for local files
- `make_playlist_id(folder: pathlib.Path) -> str` — `SHA1` of the canonical absolute
  folder path string (UTF-8 encoded)
- `load_m3u(m3u_path: pathlib.Path) -> list[pathlib.Path]` — parse M3U/M3U8 file,
  return list of resolved file paths
- `save_m3u(m3u_path: pathlib.Path, file_paths: list[pathlib.Path])` — write M3U8
  (UTF-8, `#EXTM3U` header, `#EXTINF` lines with duration and title from tags)
- `play_local_file(file_path: pathlib.Path)` — open file with `os.startfile()` on
  Windows, `subprocess.run(['open', ...])` on macOS, `xdg-open` on Linux
